Classes

Class Description Methods
LuaApi Interface between the program and Lua initialize, version?, good?, interpret, result, close

Class LuaApi Methods

Method Argument 1 Return Description
initialize None None Initializes the interface
version? None The Lua version of the interpreter in the format "Lua x.x.x" Returns the Lua version of the using interpreter.
good? None Returns true if the interface is operational, false otherwise. Checks if the interface is operational.
interpret The Lua program code to be interpreted. The result of the interpretation Interpret Lua code
result None The last result of the interpretation Returns the last result of the interpretation
close None Two, the current Interpreter Type Close the interface

Interpreter types

0 - local interpreter
1 - Online interpreter
2 - No interpreter found / interface closed

The online interpreter is from www.lua.org/cgi-bin/demo.

They can edit the interpreter type (eg LuaApi.type = 0).
Attention: Errors may occur. For example, if, after closing the interface, you switch the interpreter type and then try to interpret something.

If the local interpreter (Type 0) is used, it will create a tempolary file called "lua_api-in.txt" to execute.
This is deleted when closing the interface.

Example

require "luaapi"

lua = LuaApi.new
puts lua.version?
if lua.good?
  puts lua.interpret "print('test')"
  lua.close
end
